서론

과학자들은 엄격한 방법과 주의 깊은 관찰을 통해 질문에 답하고자 한다. 현장 노트, 설문조사, 실험 등에서 수집된 이러한 관찰 결과들이 통계적 조사의 근간을 이루며, 이를 데이터(data)라고 부른다.

통계학(statistics)은 데이터를 가장 잘 수집하고, 분석하며, 결론을 도출하는 방법을 연구하는 학문이다. 이 첫 번째 장에서는 데이터의 속성과 수집 방법 모두에 초점을 맞춘다.

1.1 사례 연구: 뇌졸중 예방을 위한 스텐트 사용

이 절에서는 통계학의 고전적인 과제인 의료 치료의 효능 평가를 소개한다. 뇌졸중 위험이 있는 환자들을 치료하는 데 있어 스텐트(stent)의 효과를 연구한 실험을 살펴볼 것이다.

핵심 연구 질문: 스텐트 사용이 뇌졸중 위험을 줄이는가?

연구자들은 위험에 처한 451명의 환자를 대상으로 실험을 수행하였다. 각 지원자 환자는 무작위로 두 집단 중 하나에 배정되었다:

치료군 (Treatment group)

스텐트와 의료 관리를 받은 환자군. 의료 관리에는 약물 투여, 위험 요인 관리, 생활 방식 개선 지원이 포함되었다. (224명)

대조군 (Control group)

치료군과 동일한 의료 관리를 받았지만, 스텐트는 받지 않은 환자군. (227명)

요약 통계량 (Summary Statistics)

요약 통계량은 대량의 데이터를 요약하는 단일 숫자이다. 1년 후 연구의 주요 결과는 다음과 같다:

집단 뇌졸중 환자 수 총 환자 수 뇌졸중 비율
치료군 (스텐트) 45 224 20%
대조군 28 227 12%
놀라운 결과: 치료군에서 추가로 8%의 환자가 뇌졸중을 겪었다! 이것은 의사들의 예상(스텐트가 뇌졸중 발생률을 줄일 것)과 정반대되는 결과이다.
예제 1.1 뇌졸중 비율 계산하기
치료군 224명 환자 중 45명이 첫 해 말까지 뇌졸중을 겪었다. 첫 해 말까지 뇌졸중을 겪은 치료군 환자의 비율을 계산하라.
풀이 보기
비율 = 45 / 224 = 0.2009 ≈ 20%

따라서 치료군에서 약 20%의 환자가 첫 해 안에 뇌졸중을 겪었다.

# 치료군 뇌졸중 비율 계산 treatment_stroke = 45 treatment_total = 224 control_stroke = 28 control_total = 227 # 비율 계산 treatment_rate = treatment_stroke / treatment_total control_rate = control_stroke / control_total difference = treatment_rate - control_rate print("=== 스텐트 연구 결과 분석 ===") print(f"치료군 뇌졸중 비율: {treatment_rate:.4f} ({treatment_rate*100:.2f}%)") print(f"대조군 뇌졸중 비율: {control_rate:.4f} ({control_rate*100:.2f}%)") print(f"차이: {difference:.4f} ({difference*100:.2f}%p)") print(f"\n결론: 치료군이 오히려 {difference*100:.1f}%p 더 높은 뇌졸중 발생률을 보임")

🔍 새로운 시각: 연구 결과의 역설과 편향 없는 관찰의 중요성

이 스텐트 연구는 통계학의 핵심 가치를 보여주는 완벽한 사례이다. 의사들과 연구자들은 스텐트가 뇌졸중 예방에 도움이 될 것이라는 사전 기대(prior expectation)를 가지고 있었다.

그러나 데이터는 기대와 정반대의 결과를 보여주었다. 이것이 바로 무작위 대조 실험(randomized controlled trial)의 힘이다.

역사적으로 많은 의학적 치료법들이 "상식적으로 효과가 있을 것 같다"는 이유로 사용되었다가, 엄격한 통계적 검증을 통해 효과가 없거나 심지어 해롭다는 것이 밝혀졌다:

  • 방혈 요법: 수 세기 동안 다양한 질병의 치료법으로 사용되었으나, 실제로는 환자를 약화시켰다
  • 초기 호르몬 대체 요법: 심장 질환 예방에 효과적일 것으로 기대되었으나, 일부 환자군에서 오히려 위험을 증가시켰다

교훈: "상식"이나 "전문가 의견"보다 데이터를 신뢰하라. 통계학의 역할은 우리의 편견을 확인하는 것이 아니라, 현실을 객관적으로 드러내는 것이다.

1.2 데이터 기초 (Data Basics)

데이터의 효과적인 조직과 설명은 대부분의 분석에서 첫 번째 단계이다. 이 절에서는 데이터를 조직하기 위한 데이터 행렬과 다양한 형태의 데이터에 대한 용어를 소개한다.

1.2.1 관측값, 변수, 데이터 행렬

케이스 (Case) / 관측 단위 (Observational Unit)

데이터 행렬의 각 행을 나타내며, 단일 관측 대상을 의미한다.

변수 (Variable)

각 케이스에 대한 특성을 나타내며, 데이터 행렬의 열에 해당한다.

데이터 행렬 (Data Matrix)

데이터를 조직하는 편리하고 일반적인 방법. 각 행은 고유한 케이스에, 각 열은 변수에 해당한다.

1.2.2 변수의 유형 (Types of Variables)

변수 유형 세부 유형 설명 예시
수치형 (Numerical) 연속 (Continuous) 연속적인 실수 값 키, 몸무게, 온도
이산 (Discrete) 정수 값만 취함 인구 수, 형제 수
범주형 (Categorical) 명목형 (Nominal) 순서 없는 범주 성별, 혈액형, 국가
순서형 (Ordinal) 순서가 있는 범주 교육 수준, 만족도
예제 1.5 변수 유형 분류하기
통계학 수업에서 학생들에 대한 데이터를 수집하였다. 각 학생에 대해 세 가지 변수가 기록되었다: 형제자매 수, 학생 키, 이전에 통계학 수업을 수강한 적이 있는지 여부. 각 변수를 연속 수치형, 이산 수치형, 또는 범주형으로 분류하라.
풀이 보기
  • 형제자매 수: 개수이므로 이산 수치형
  • 학생 키: 연속적으로 변하므로 연속 수치형
  • 통계학 수강 여부: 두 범주(수강/미수강)로 분류되므로 범주형
import pandas as pd # 학생 데이터 예시 students = pd.DataFrame({ '형제자매_수': [2, 1, 3, 0, 2], # 이산 수치형 '키_cm': [165.5, 178.2, 162.3, 180.0, 170.8], # 연속 수치형 '통계학_수강_여부': ['예', '아니오', '예', '예', '아니오'] # 범주형 }) print("=== 학생 데이터셋 ===") print(students) print("\n=== 변수 유형 분류 ===") print("- 형제자매_수: 이산 수치형 (개수, 정수만 가능)") print("- 키_cm: 연속 수치형 (실수 값, 연속적 변화)") print("- 통계학_수강_여부: 범주형 (두 범주)") print("\n=== 기술통계 ===") print(f"형제자매 수 평균: {students['형제자매_수'].mean():.2f}") print(f"키 평균: {students['키_cm'].mean():.2f}cm") print(f"통계학 수강 비율: {(students['통계학_수강_여부'] == '예').mean()*100:.1f}%")

1.2.3 변수 간의 관계

연관 (Associated)

두 변수 사이에 어떤 관계가 있을 때, 이들은 연관되어 있다고 한다. 음의 연관(한 변수 증가 시 다른 변수 감소) 또는 양의 연관(함께 증가/감소)이 있을 수 있다.

독립 (Independent)

두 변수 사이에 명백한 관계가 없다면 독립적이라고 한다.

중요: 한 쌍의 변수는 연관이거나 독립 둘 중 하나이다. 어떤 변수 쌍도 연관이면서 동시에 독립일 수 없다.

1.2.4 설명 변수와 반응 변수

설명 변수 (Explanatory Variable)

다른 변수에 인과적으로 영향을 미칠 수 있다고 의심되는 변수. 독립 변수라고도 한다.

반응 변수 (Response Variable)

설명 변수의 영향을 받는다고 가정되는 변수. 종속 변수라고도 한다.

주의: 변수에 이런 방식으로 레이블을 붙이는 행위가 인과 관계의 존재를 보장하지 않는다. 인과 관계를 확인하기 위해서는 실험이 필요하다.

1.2.5 관찰 연구와 실험

특성 관찰 연구 (Observational Study) 실험 (Experiment)
처치 할당 연구자가 간섭하지 않음 연구자가 처치를 할당
인과 추론 불가능 (연관만 확인 가능) 가능 (무작위 배정 시)
예시 설문조사, 의료 기록 검토 임상 시험, A/B 테스트
연관 ≠ 인과: 일반적으로, 연관은 인과를 의미하지 않으며, 인과는 무작위 실험에서만 추론될 수 있다.

🔍 새로운 시각: 인과 추론의 근본적 문제

인과 추론(causal inference)은 통계학에서 가장 도전적이면서도 중요한 주제 중 하나이다. "상관은 인과가 아니다"라는 문구는 통계학의 가장 중요한 경고이다.

왜 관찰 연구에서 인과 관계를 주장할 수 없는가? 그 이유는 혼란 변수(confounding variable)의 존재 때문이다.

예시: 아이스크림 판매량과 익사 사고 수 사이에 강한 양의 상관관계가 있다. 그러나 아이스크림이 익사를 야기하는 것은 아니다. 두 변수 모두 기온이라는 제3의 변수에 의해 영향을 받는다.

무작위 실험이 이 문제를 해결하는 이유는 무작위 배정이 알려진 혼란 변수와 알려지지 않은 혼란 변수를 모두 균형있게 분배하기 때문이다.

실습 변수 유형과 관계 분석
county 데이터셋을 시뮬레이션하여 변수 유형을 분류하고, 변수 간 관계를 분석한다.
import pandas as pd import numpy as np # county 데이터셋 시뮬레이션 np.random.seed(123) n_counties = 100 county_data = pd.DataFrame({ 'name': [f'County_{i}' for i in range(n_counties)], 'state': np.random.choice(['CA', 'TX', 'FL', 'NY', 'PA'], n_counties), 'pop': np.random.randint(10000, 500000, n_counties), 'poverty': np.random.uniform(5, 25, n_counties), 'homeownership': np.random.uniform(50, 85, n_counties), 'multi_unit': np.random.uniform(5, 50, n_counties), 'metro': np.random.choice(['yes', 'no'], n_counties) }) # 음의 상관관계 부여 (homeownership과 multi_unit) county_data['homeownership'] = 85 - 0.5 * county_data['multi_unit'] + np.random.normal(0, 5, n_counties) print("=== 변수 유형 분류 ===") print("\n수치형 변수:") print(" - pop: 이산형 (인구 수)") print(" - poverty: 연속형 (빈곤율)") print(" - homeownership: 연속형 (자가 소유율)") print(" - multi_unit: 연속형 (다세대 비율)") print("\n범주형 변수:") print(f" - state: {county_data['state'].nunique()}개 수준") print(f" - metro: {county_data['metro'].nunique()}개 수준") # 상관관계 분석 corr = np.corrcoef(county_data['multi_unit'], county_data['homeownership'])[0, 1] print(f"\n=== 변수 간 관계 ===") print(f"homeownership vs multi_unit 상관계수: {corr:.4f}") if corr < -0.3: print("해석: 강한 음의 연관 (다세대 비율↑ → 자가소유율↓)") elif corr < 0: print("해석: 약한 음의 연관") else: print("해석: 양의 연관 또는 독립")
1.3 표본추출 원리와 전략

1.3.1 모집단과 표본

모집단 (Population)

연구 질문이 참조하는 전체 대상 집합. 모집단의 모든 케이스에 대해 데이터를 수집하는 것은 종종 비용이 많이 든다.

표본 (Sample)

모집단에서 선택된 케이스의 부분집합. 표본 데이터를 사용하여 모집단에 대한 추정치를 제공한다.

1.3.2 일화적 증거

일화적 증거 (Anecdotal Evidence): 무분별하게 수집된 데이터에 주의하라. 그러한 증거는 사실이고 검증 가능할 수 있지만, 특이한 케이스만을 나타낼 수 있다.

일화적 증거는 일반적으로 그 특이한 특성 때문에 기억에 남는 비정상적인 케이스로 구성된다. 모집단을 대표하는 많은 케이스의 표본을 조사해야 한다.

🔍 새로운 시각: 인지 편향과 일화적 증거의 심리학

왜 우리는 일화적 증거에 쉽게 설득되는가? 이것은 인지 편향(cognitive bias)과 관련이 있다.

  • 가용성 휴리스틱: 쉽게 떠오르는 예시에 더 큰 중요성을 부여하는 경향
  • 확증 편향: 자신의 기존 믿음을 확인하는 정보를 선택적으로 찾고 기억하는 경향
  • 대표성 휴리스틱: 작은 표본이 전체 모집단을 대표한다고 가정하는 경향

통계학은 이러한 인지적 함정을 피하기 위해 체계적인 데이터 수집과 객관적인 분석 방법을 사용한다.

1.3.3 표본추출의 편향

단순 무작위 표본 (Simple Random Sample)

모집단의 각 케이스가 포함될 동등한 기회를 가지며, 한 케이스가 표본에 포함되어 있다는 것이 어떤 다른 케이스가 포함되어 있는지에 대한 정보를 제공하지 않는 표본.

무응답 편향 (Non-response Bias)

무작위로 선택된 사람들 중 일부만 실제로 응답할 때, 결과가 전체 모집단을 대표하는지 불분명해지는 편향.

편의 표본 (Convenience Sample)

쉽게 접근할 수 있는 개인이 표본에 포함될 가능성이 더 높은 표본. 모집단을 대표하지 않을 수 있다.

1.3.5 네 가지 표본추출 방법

방법 설명 장점 단점
단순 무작위 모집단에서 동등한 확률로 선택 가장 기본적, 편향 없음 비용이 많이 들 수 있음
층화 모집단을 층으로 나눈 후 각 층에서 표본 추출 각 하위집단 대표 보장 분석이 복잡함

예제 1.15 말라리아 연구: 표본추출 방법 선택
인도네시아 농촌의 열대 밀림 지역에서 말라리아 발생률을 추정하려 한다. 30개의 마을이 있으며, 각각은 다른 것과 거의 유사하다. 목표는 150명을 말라리아 검사하는 것이다. 어떤 표본추출 방법을 사용해야 하는가?
풀이 보기

다단계 표본추출이 가장 적합하다. 마을의 절반(15개)을 무작위로 선택한 다음 각 마을에서 10명을 무작위로 선택한다. 이것은 단순 무작위 표본에 비해 데이터 수집 비용을 상당히 줄이면서도 신뢰할 수 있는 정보를 제공한다.

import numpy as np np.random.seed(42) # 30개 마을, 각 마을 약 100명 n_villages = 30 n_per_village = 100 target_sample = 150 print("=== 말라리아 연구: 표본추출 방법 비교 ===") # 1. 단순 무작위 표본추출 print("\n1. 단순 무작위 표본추출 (SRS)") villages_visited_srs = set() for _ in range(target_sample): village = np.random.randint(0, n_villages) villages_visited_srs.add(village) print(f" 방문해야 할 마을 수: ~{len(villages_visited_srs)}개") print(f" 비용: 매우 높음 (거의 모든 마을 방문)") # 2. 군집 표본추출 print("\n2. 군집 표본추출") n_clusters = 2 # 2개 마을 선택 sample_cluster = n_per_village * n_clusters print(f" 선택된 마을 수: {n_clusters}개") print(f" 총 표본: {sample_cluster}명 (마을 전체)") print(f" 비용: 낮음 (2개 마을만 방문)") # 3. 다단계 표본추출 (권장) print("\n3. 다단계 표본추출 (권장)") n_clusters_multi = 15 # 마을의 절반 선택 sample_per_cluster = 10 # 각 마을에서 10명 print(f" 선택된 마을 수: {n_clusters_multi}개") print(f" 마을당 표본: {sample_per_cluster}명") print(f" 총 표본: {n_clusters_multi * sample_per_cluster}명") print(f" 비용: 중간 (SRS보다 낮고, 다양성 확보)") print("\n결론: 다단계 표본추출이 비용과 대표성 사이의 최적의 균형을 제공함")

🔍 새로운 시각: 표본추출 방법의 실제 적용

실제 세계에서 표본추출 방법의 선택은 비용, 정확성, 실현 가능성의 균형을 맞춰야 한다.

  • 여론조사: 층화 표본추출 - 지역, 연령대, 성별로 층화
  • 의료 연구: 군집 표본추출 - 환자가 집중된 병원을 군집으로 설정
  • 교육 연구: 다단계 표본추출 - 시·도 → 학교 → 학급 → 학생 순으로 선택
1.4 실험 (Experiments)

연구자가 케이스에 처치를 할당하는 연구를 실험(experiment)이라고 한다. 이 할당이 무작위화를 포함할 때, 이를 무작위 실험(randomized experiment)이라고 한다.

1.4.1 실험 설계의 원칙

통제 (Control)

연구자는 실험에서 잠재적으로 혼란을 주는 변수들을 통제하려고 노력해야 한다.

무작위화 (Randomize)

환자를 무작위로 처치 집단에 배정함으로써 통제할 수 없는 변수들을 그룹 간에 균형있게 배분한다.

반복 (Replicate)

대상들이 많을수록 개별적 변동이 평균화되어 처치 효과의 추정치가 더 정확해진다.

블로킹 (Block)

처치에 대한 반응에 영향을 미칠 것으로 알려진 변수가 있을 때, 먼저 개인을 유사한 그룹(블록)으로 나눈 후 각 블록 내에서 무작위화한다.

🔍 새로운 시각: 피셔의 세 가지 실험 설계 원칙

현대 실험 설계는 20세기 초 영국 통계학자 로널드 피셔(Ronald A. Fisher)의 연구에서 비롯되었다.

  • 무작위화: "처치와 실험 단위 간의 관계를 우연에 맡긴다" - 이것이 비교를 공정하게 만든다.
  • 반복: "동일한 처치에 대한 여러 관측값" - 이것이 추정의 정밀도를 높인다.
  • 블로킹: "무엇을 통제할 수 있으면 블록화하고, 통제할 수 없으면 무작위화하라" - 이것이 알려진 변동 원인을 제거한다.

1.4.2 인체 실험에서 편향 줄이기

위약 (Placebo)

가짜 처치. 효과적인 위약은 연구를 진정으로 맹검으로 만드는 핵심이다. 종종 위약은 환자에게 약간의 실제 개선을 가져오며, 이를 위약 효과라고 한다.

맹검 (Blind)

연구자들이 환자에게 그들의 처치에 대해 알리지 않을 때, 그 연구는 맹검이라고 한다.

이중 맹검 (Double-blind)

환자와 환자와 상호작용하는 의사나 연구자 모두 누가 처치를 받고 있고 누가 받지 않는지 모르는 설정.

실습 임상 시험 시뮬레이션: 혈압 약 효과 검증
블로킹을 사용한 무작위 실험을 시뮬레이션하여 새로운 혈압 약의 효과를 검증한다.
import numpy as np import pandas as pd from scipy import stats np.random.seed(42) # 실험 설계 파라미터 n_patients = 200 # 환자 특성 생성 (블로킹을 위한 위험 수준) risk_levels = np.random.choice(['low', 'high'], n_patients, p=[0.6, 0.4]) # 블로킹을 사용한 무작위 배정 def blocked_randomization(risk_levels): assignments = np.empty(len(risk_levels), dtype=object) for risk in ['low', 'high']: risk_idx = np.where(risk_levels == risk)[0] n_risk = len(risk_idx) # 각 블록 내에서 절반씩 무작위 배정 treatment_idx = np.random.choice(risk_idx, n_risk // 2, replace=False) assignments[treatment_idx] = 'treatment' control_idx = np.setdiff1d(risk_idx, treatment_idx) assignments[control_idx] = 'control' return assignments group_assignment = blocked_randomization(risk_levels) # 기저 혈압 baseline_bp = np.where(risk_levels == 'high', np.random.normal(160, 10, n_patients), np.random.normal(145, 10, n_patients)) # 처치 효과 true_treatment_effect = -15 # 치료군: 평균 15mmHg 감소 placebo_effect = -5 # 대조군: 5mmHg 감소 (위약 효과) bp_change = np.where(group_assignment == 'treatment', np.random.normal(true_treatment_effect, 8, n_patients), np.random.normal(placebo_effect, 8, n_patients)) # 데이터프레임 생성 trial_data = pd.DataFrame({ 'risk_level': risk_levels, 'group': group_assignment, 'baseline_bp': baseline_bp, 'bp_change': bp_change }) print("=== 임상 시험 시뮬레이션: 혈압 약 효과 검증 ===") # 1. 블로킹 확인 print("\n1. 블로킹 확인 (위험 수준별 집단 분포):") print(pd.crosstab(trial_data['risk_level'], trial_data['group'])) # 2. 처치 효과 분석 print("\n2. 처치 효과 분석:") treatment_change = trial_data[trial_data['group'] == 'treatment']['bp_change'] control_change = trial_data[trial_data['group'] == 'control']['bp_change'] print(f" 치료군 혈압 변화: {treatment_change.mean():.2f} ± {treatment_change.std():.2f} mmHg") print(f" 대조군 혈압 변화: {control_change.mean():.2f} ± {control_change.std():.2f} mmHg") # 3. 통계적 검정 t_stat, p_value = stats.ttest_ind(treatment_change, control_change) effect_size = treatment_change.mean() - control_change.mean() print(f"\n3. 통계적 검정 결과:") print(f" 처치 효과 (차이): {effect_size:.2f} mmHg") print(f" t-통계량: {t_stat:.3f}") print(f" p-값: {p_value:.6f}") if p_value < 0.05: print(f"\n결론: 통계적으로 유의한 차이 있음 (위약 대비 {abs(effect_size):.1f}mmHg 추가 감소)")

연습문제 1.1 편두통과 침술 (Migraine and Acupuncture)
편두통 치료를 위한 침술의 효과를 연구하였다. 89명의 여성을 두 집단에 무작위 배정하였다: 치료군(43명)은 편두통 치료용 침술을, 대조군(46명)은 위약 침술(비경혈 위치)을 받았다. 24시간 후 결과는 다음과 같다:

통증 없음통증 있음합계
치료군103343
대조군24446

(a) 치료군에서 통증이 없는 환자의 비율은?
(b) 대조군에서 통증이 없는 환자의 비율은?
(c) 어느 집단에서 더 높은 통증 완화율을 보이는가?
(d) 관찰된 차이에 대한 다른 가능한 설명은?
풀이 보기

(a) 치료군 통증 완화율: 10/43 = 23.3%

(b) 대조군 통증 완화율: 2/46 = 4.3%

(c) 치료군에서 더 높은 비율의 환자가 통증이 없었다 (23.3% > 4.3%)

(d) 관찰된 차이가 단순히 우연(chance variation) 때문일 가능성이 있다. 무작위 변동으로 인해 한 집단에서 더 좋은 결과가 나타날 수 있다.

import numpy as np from scipy.stats import chi2_contingency # 데이터 정의 treatment_pain_free = 10 treatment_total = 43 control_pain_free = 2 control_total = 46 # 비율 계산 treatment_rate = treatment_pain_free / treatment_total control_rate = control_pain_free / control_total difference = treatment_rate - control_rate print("=== 편두통 침술 연구 결과 분석 ===") print(f"(a) 치료군 통증 완화율: {treatment_rate:.4f} ({treatment_rate*100:.2f}%)") print(f"(b) 대조군 통증 완화율: {control_rate:.4f} ({control_rate*100:.2f}%)") print(f"(c) 차이: {difference:.4f} ({difference*100:.2f}%p)") print(f" 치료군이 {difference*100:.1f}%p 더 높은 통증 완화율을 보임") # 카이제곱 검정 observed = np.array([[10, 33], [2, 44]]) chi2, p_value, dof, expected = chi2_contingency(observed) print(f"\n(d) 카이제곱 검정 결과:") print(f" 카이제곱 통계량: {chi2:.4f}") print(f" p-값: {p_value:.4f}") if p_value < 0.05: print(" p < 0.05: 통계적으로 유의한 차이가 있음") else: print(" p >= 0.05: 차이가 우연에 의한 것일 수 있음")
연습문제 1.9 피셔의 붓꽃 (Fisher's Irises)
로널드 피셔 경은 세 종의 붓꽃(setosa, versicolor, virginica)에서 꽃받침 길이와 너비, 꽃잎 길이와 너비를 포함하는 데이터셋을 다루었다. 각 종에서 50송이의 꽃이 데이터셋에 있다.

(a) 데이터에 몇 개의 케이스가 포함되었는가?
(b) 몇 개의 수치 변수가 데이터에 포함되었는가?
(c) 몇 개의 범주형 변수가 데이터에 포함되었는가?
풀이 보기

(a) 50 × 3 = 150개의 케이스

(b) 4개의 연속 수치 변수: 꽃받침 길이, 꽃받침 너비, 꽃잎 길이, 꽃잎 너비

(c) 1개의 범주형 변수: 종 (species) - setosa, versicolor, virginica 세 수준

from sklearn.datasets import load_iris import pandas as pd # 붓꽃 데이터 로드 iris = load_iris() df = pd.DataFrame(iris.data, columns=iris.feature_names) df['species'] = [iris.target_names[i] for i in iris.target] print("=== 피셔의 붓꽃 데이터셋 ===") print(f"\n(a) 케이스 수: {len(df)}개 (50개 × 3종)") print("\n(b) 수치 변수 (4개, 모두 연속형):") for var in iris.feature_names: print(f" - {var}") print("\n(c) 범주형 변수 (1개):") print(f" - species: {', '.join(iris.target_names)}") # 기술통계 print("\n=== 종별 평균 ===") print(df.groupby('species')[iris.feature_names].mean().round(2))
