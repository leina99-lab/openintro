2.1.1 짝지어진 데이터를 위한 산점도(Scatterplot)

산점도(scatterplot)는 두 개의 수치형 변수에 대한 데이터를 케이스별로 보여주는 그래프이다. 각 점은 하나의 케이스를 나타낸다. 그림 2.1은 loan50 데이터셋에서 차용인의 총소득(total income)과 대출금액(loan amount)을 비교하는 산점도이다.

그림 2.1: loan50 데이터셋의 총소득 대 대출금액 산점도
예제 2.1 비선형적 관계 탐색
[문제] 그림 2.2는 3,142개 카운티에 대한 가구 중위소득과 빈곤율의 산점도이다. 이 변수들 간의 관계에 대해 무엇을 말할 수 있는가?
그림 2.2: county 데이터셋의 빈곤율 대 가구 중위소득 산점도
💡 정답 확인
이 관계는 점선이 강조하듯이 명백히 비선형적이다. 빈곤율이 증가함에 따라 가구 중위소득이 감소하되, 그 감소폭이 점차 줄어드는 곡선 형태를 보인다. 이는 단순 직선으로 설명할 수 없는 관계이다.
import numpy as np import matplotlib.pyplot as plt # county 데이터 시뮬레이션 np.random.seed(123) n_counties = 500 poverty_rate = np.random.beta(2, 8, n_counties) * 50 median_income = 100000 * np.exp(-0.03 * poverty_rate) + np.random.normal(0, 10000, n_counties) median_income = np.clip(median_income, 10000, 130000) plt.figure(figsize=(10, 6)) plt.scatter(poverty_rate, median_income/1000, alpha=0.4, s=10, c='steelblue') # 추세선 x_fit = np.linspace(0, 50, 100) y_fit = 100 * np.exp(-0.03 * x_fit) plt.plot(x_fit, y_fit, 'r--', linewidth=2, label='추세선 (비선형)') plt.xlabel('빈곤율 (퍼센트)') plt.ylabel('가구 중위소득 (천 달러)') plt.title('그림 2.2: 빈곤율과 가구 중위소득의 관계') plt.legend() plt.grid(True, alpha=0.3) plt.show() corr = np.corrcoef(poverty_rate, median_income)[0, 1] print(f"피어슨 상관계수: {corr:.4f}")
새로운 시각: 산점도의 철학적 의미

산점도는 단순한 시각화 도구 이상의 의미를 가진다. 19세기 영국의 통계학자 프랜시스 골턴(Francis Galton)이 처음 체계적으로 사용한 이후, 산점도는 변수 간 관계를 탐색하는 가장 강력한 도구 중 하나로 자리잡았다.

핵심 통찰:

1. 상관관계 ≠ 인과관계: 산점도에서 강한 패턴이 보인다고 해서 한 변수가 다른 변수를 "일으킨다"고 결론 내릴 수 없다.

2. 심슨의 역설 주의: 전체 데이터에서 보이는 관계가 하위 그룹에서는 반대로 나타날 수 있다.

3. 이상치의 영향: 극단적인 점 하나가 전체 관계의 인상을 바꿀 수 있으므로 항상 이상치를 확인해야 한다.

연습문제 2.1 다양한 관계 유형 탐색하기
[미션] 아래 코드에서 noise_level 값을 변경하여 (0.5, 2, 5, 10 등) 노이즈가 산점도 패턴에 미치는 영향을 관찰하라. 또한 relationship을 'negative' 또는 'quadratic'으로 변경해 보라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) n = 100 # ★ 아래 값을 변경해 보라 ★ noise_level = 2 # 노이즈 수준: 0.5, 2, 5, 10 relationship = 'positive' # 관계 유형: 'positive', 'negative', 'quadratic' x = np.random.uniform(0, 10, n) if relationship == 'positive': y = 2 * x + np.random.normal(0, noise_level, n) title = '양의 선형 관계' elif relationship == 'negative': y = -1.5 * x + 20 + np.random.normal(0, noise_level, n) title = '음의 선형 관계' elif relationship == 'quadratic': y = (x - 5)**2 + np.random.normal(0, noise_level, n) title = '비선형 관계 (이차)' else: y = np.random.uniform(0, 20, n) title = '관계 없음' plt.figure(figsize=(8, 5)) plt.scatter(x, y, alpha=0.6, c='steelblue') plt.xlabel('X'); plt.ylabel('Y') plt.title(f'{title} (노이즈 수준 = {noise_level})') plt.grid(True, alpha=0.3) plt.show() corr = np.corrcoef(x, y)[0, 1] print(f"상관계수 r = {corr:.4f}") print(f"노이즈가 클수록 상관계수의 절댓값이 감소함을 확인할 수 있다.")
💡 해석 가이드
노이즈 수준이 낮을수록(예: 0.5) 점들이 추세선에 가까이 모여 상관계수의 절댓값이 1에 가까워진다. 노이즈 수준이 높을수록(예: 10) 점들이 넓게 퍼져 상관계수가 0에 가까워진다. 비선형 관계('quadratic')에서는 피어슨 상관계수가 낮게 나타나지만, 실제로는 강한 관계가 존재한다. 이는 피어슨 상관계수가 선형적 관계만 측정하기 때문이다.
2.1.2 점도표와 평균(Dot Plot & Mean)

단일 변수만 관심 대상일 때, 점도표(dot plot)가 가장 기본적인 시각화를 제공한다. 점도표는 일변량 산점도이다.

그림 2.3: loan50 데이터의 이자율 점도표 (삼각형 = 평균 11.73%)
그림 2.4: loan50 데이터의 누적 점도표
정의: 평균(Mean)

표본 평균은 관측값의 합을 관측치 수로 나누어 계산한다:

x̄ = (x₁ + x₂ + … + xₙ) / n

여기서 x₁, x₂, …, xₙ은 n개의 관측값을 나타낸다. 평균은 분포의 균형점으로 이해할 수 있다.

예제 2.7 평균이 비교에 유용한 사례
[문제] 신약(500명, 발작 200건)과 표준 약(1000명, 발작 300건)의 원시 수치만 비교하면 신약이 좋아 보인다. 그러나 평균을 사용하면 결론이 달라진다. 왜 그런가?
💡 정답 확인
환자당 평균 발작 수를 구하면, 신약은 200/500 = 0.4이고, 표준 약은 300/1000 = 0.3이다. 따라서 표준 약이 환자당 평균 발작 수가 더 낮아, 실제로 표준 약이 더 효과적이다. 원시 카운트만 비교하면 불균형한 그룹 크기로 인해 잘못된 결론을 도출할 수 있다.
새로운 시각: 평균의 기하학적 해석

평균을 "균형점"으로 이해하는 것은 매우 직관적이다. 물리학적으로, 데이터 점들을 막대 위의 무게라고 생각하면, 평균은 막대가 수평으로 균형을 이루는 지점이다.

지렛대 원리: 평균에서 멀리 있는 값은 더 큰 "토크"를 행사하여 평균을 자신 쪽으로 끌어당긴다. 빌 게이츠가 방에 들어오면 방 안 사람들의 평균 자산은 급격히 증가한다. 중위수는 "위치적 중심"이고, 평균은 "질량적 중심"이다.

연습문제 2.2 단순 평균 vs 가중 평균 비교
[미션] 아래 코드에서 populations와 per_capita_incomes의 값을 변경하여, 인구가 불균형할 때 단순 평균과 가중 평균의 차이가 얼마나 커지는지 확인하라.
import numpy as np # ★ 아래 값을 변경해 보라 ★ # 예: 인구가 극단적으로 다른 경우 [100, 100, 5000000] populations = np.array([5000, 50000, 5000000]) per_capita_incomes = np.array([20000, 30000, 55000]) # 단순 평균 (잘못된 방법) simple_mean = np.mean(per_capita_incomes) print(f"단순 평균 (잘못된 방법): ${simple_mean:,.0f}") # 가중 평균 (올바른 방법) weighted_mean = np.average(per_capita_incomes, weights=populations) print(f"가중 평균 (올바른 방법): ${weighted_mean:,.0f}") print(f"\n차이: ${weighted_mean - simple_mean:,.0f}") print(f"차이 비율: {(weighted_mean - simple_mean) / simple_mean * 100:.1f}%") print("\n해석: 인구가 큰 카운티의 소득이 가중 평균에 더 큰 영향을 미친다.")
💡 해석 가이드
인구가 5,000,000인 카운티의 1인당 소득이 $55,000이므로, 가중 평균은 $55,000에 가까워진다. 반면 단순 평균은 세 카운티를 동등하게 취급하여 $35,000이 된다. 이처럼 관측 단위의 크기가 다를 때는 가중 평균을 사용해야 정확한 전체 평균을 얻을 수 있다.
2.1.3 히스토그램과 형태(Histogram & Shape)

점도표는 각 관측치의 정확한 값을 보여주지만, 큰 데이터셋에서는 읽기 어려워질 수 있다. 히스토그램(histogram)은 값을 구간(bin)에 할당하고 각 구간의 빈도를 막대로 표현한다. 히스토그램은 데이터 분포의 형태(shape)를 이해하는 데 특히 편리하다.

그림 2.6: 이자율 히스토그램 (오른쪽으로 강하게 치우침)
정의: 분포의 형태

데이터가 오른쪽으로 길게 늘어지면 오른쪽으로 치우침(right skewed), 왼쪽으로 길게 늘어지면 왼쪽으로 치우침(left skewed), 양쪽 방향으로 대략 동등하게 늘어지면 대칭적(symmetric)이라 한다.

또한 히스토그램은 최빈값(mode), 즉 분포에서 뚜렷한 봉우리를 식별하는 데 사용된다. 봉우리 수에 따라 단봉(unimodal), 이봉(bimodal), 다봉(multimodal)으로 분류한다.

그림 2.7: 단봉, 이봉, 다봉 분포의 히스토그램
예제 2.11 분포의 최빈값 수 판별
[문제] 그림 2.6의 이자율 히스토그램은 단봉인가, 이봉인가, 다봉인가?
💡 정답 확인
단봉(unimodal)이다. 오직 하나의 뚜렷한 봉우리가 존재한다. 접두사 uni-는 1을 의미하며(unicycle = 외바퀴 자전거), bi-는 2를 의미한다(bicycle = 두 바퀴 자전거).
새로운 시각: 히스토그램의 구간 폭 선택

히스토그램을 만들 때 가장 중요한 결정 중 하나는 구간(bin)의 폭을 선택하는 것이다. 구간이 너무 좁으면 노이즈가 많아지고, 너무 넓으면 중요한 특징을 놓칠 수 있다.

스터지스의 규칙(Sturges' Rule): k = 1 + 3.322 × log₁₀(n)

프리드먼-디아코니스 규칙: 구간 폭 = 2 × IQR / n^(1/3)

연습문제 2.3 구간 수(bins)와 분포 형태의 관계
[미션] 아래 코드에서 n_bins를 5, 15, 30, 100으로 변경하면서 히스토그램의 모양이 어떻게 달라지는지 관찰하라. 또한 dist_type을 변경하여 다양한 분포를 확인해 보라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) # ★ 아래 값을 변경해 보라 ★ n_bins = 20 # 구간 수: 5, 15, 30, 100 dist_type = 'right_skew' # 분포 유형: 'right_skew', 'left_skew', 'symmetric', 'bimodal' n = 500 if dist_type == 'right_skew': data = np.random.exponential(10, n) label = '오른쪽으로 치우침 (Right-skewed)' elif dist_type == 'left_skew': data = 100 - np.random.exponential(10, n) label = '왼쪽으로 치우침 (Left-skewed)' elif dist_type == 'symmetric': data = np.random.normal(50, 10, n) label = '대칭 (Symmetric)' elif dist_type == 'bimodal': data = np.concatenate([np.random.normal(30, 5, 250), np.random.normal(70, 5, 250)]) label = '이봉 (Bimodal)' plt.figure(figsize=(10, 5)) plt.hist(data, bins=n_bins, edgecolor='black', alpha=0.7, color='steelblue') plt.axvline(np.mean(data), color='red', linestyle='--', linewidth=2, label=f'평균 = {np.mean(data):.2f}') plt.axvline(np.median(data), color='green', linestyle='-', linewidth=2, label=f'중위수 = {np.median(data):.2f}') plt.xlabel('값') plt.ylabel('빈도') plt.title(f'{label} (구간 수 = {n_bins})') plt.legend() plt.show() from scipy.stats import skew print(f"치우침(skewness): {skew(data):.2f}") print("양수 = 오른쪽 치우침, 음수 = 왼쪽 치우침, 0 ≈ 대칭")
💡 해석 가이드
구간 수가 너무 적으면(5개) 분포의 세부적인 형태가 사라지고, 너무 많으면(100개) 노이즈가 과다하여 전체 패턴을 파악하기 어렵다. 15~30개 정도가 일반적으로 적절하다. 오른쪽 치우침 분포에서는 평균 > 중위수이고, 왼쪽 치우침에서는 평균 < 중위수임을 확인할 수 있다.
2.1.4 분산과 표준편차(Variance & Standard Deviation)

평균이 분포의 중심을 나타낸다면, 분산과 표준편차는 데이터가 평균으로부터 얼마나 퍼져 있는지를 측정한다.

정의: 분산과 표준편차

편차(deviation): xᵢ − x̄ (각 관측값에서 평균을 뺀 값)

분산(variance): s² = Σ(xᵢ − x̄)² / (n − 1)

표준편차(standard deviation): s = √s²

분모에 n이 아닌 n−1을 사용하는 것은 베셀 보정(Bessel's correction)이라 하며, 표본 분산이 모분산의 불편추정량이 되도록 하기 위함이다.

그림 2.8: 이자율의 평균과 표준편차 구간 시각화
그림 2.9: 같은 평균과 표준편차를 가진 매우 다른 분포들
연습문제 2.4 표준편차 직접 계산하기
[미션] 아래 data 리스트의 값을 변경하여, 데이터가 밀집될수록 표준편차가 작아지고, 퍼질수록 커지는 것을 확인하라. 예를 들어 [5,5,5,5,5]와 [1,3,5,7,9]를 비교해 보라.
import numpy as np # ★ 아래 데이터를 변경해 보라 ★ data = np.array([10.90, 9.92, 26.30, 9.43, 9.92, 6.08]) n = len(data) mean = np.mean(data) print(f"데이터: {data}") print(f"표본 크기 n = {n}") print(f"평균 x̄ = {mean:.2f}") # 단계별 계산 deviations = data - mean squared_devs = deviations ** 2 print(f"\n{'관측값':>8} {'편차':>10} {'편차²':>12}") print("-" * 35) for i in range(n): print(f"{data[i]:8.2f} {deviations[i]:10.2f} {squared_devs[i]:12.2f}") variance = np.sum(squared_devs) / (n - 1) std_dev = np.sqrt(variance) print(f"\n편차² 합계 = {np.sum(squared_devs):.2f}") print(f"분산 s² = {np.sum(squared_devs):.2f} / {n-1} = {variance:.2f}") print(f"표준편차 s = √{variance:.2f} = {std_dev:.2f}") print(f"\nNumPy 검증: s = {np.std(data, ddof=1):.2f}")
💡 해석 가이드
표준편차는 데이터가 평균으로부터 전형적으로 얼마나 떨어져 있는지를 나타낸다. 모든 값이 같으면(예: [5,5,5,5,5]) 표준편차는 0이다. 이상치(예: 26.30)가 포함되면 표준편차가 크게 증가한다. 이는 표준편차가 이상치에 민감하다는 것을 보여준다.
2.1.5 상자그림, IQR, 이상치(Box Plot, IQR, Outlier)

상자그림(box plot)은 다섯 수 요약(최솟값, Q1, 중위수, Q3, 최댓값)을 시각적으로 나타낸다. IQR(사분위범위)은 Q3 − Q1으로 정의되며, 데이터의 중간 50%가 차지하는 범위이다.

정의: 이상치 판별 규칙 (IQR 규칙)

관측값이 다음 범위를 벗어나면 이상치(outlier)로 간주한다:

Q1 − 1.5 × IQR   또는   Q3 + 1.5 × IQR

상자그림에서 수염(whisker)은 이 범위 내의 가장 극단적인 값까지 연장되며, 범위 밖의 값은 별도의 점으로 표시된다.

그림 2.10: 이자율의 점도표와 상자그림 비교
연습문제 2.5 상자그림과 이상치 탐색
[미션] 아래 outlier_value를 50, 80, 120, 200 등으로 변경하여 이상치가 상자그림과 통계량에 미치는 영향을 관찰하라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) # ★ 아래 값을 변경해 보라 ★ outlier_value = 120 # 이상치 값: 50, 80, 120, 200 # 기본 데이터 (정규분포) base_data = np.random.normal(50, 10, 49) data = np.append(base_data, outlier_value) # 통계량 계산 q1 = np.percentile(data, 25) median = np.median(data) q3 = np.percentile(data, 75) iqr = q3 - q1 lower_fence = q1 - 1.5 * iqr upper_fence = q3 + 1.5 * iqr fig, axes = plt.subplots(1, 2, figsize=(12, 5)) # 히스토그램 axes[0].hist(data, bins=20, edgecolor='black', alpha=0.7, color='steelblue') axes[0].axvline(outlier_value, color='red', linestyle='--', label=f'이상치 = {outlier_value}') axes[0].set_title('히스토그램') axes[0].legend() # 상자그림 bp = axes[1].boxplot(data, vert=True, patch_artist=True) bp['boxes'][0].set_facecolor('lightcyan') axes[1].set_title('상자그림') axes[1].set_ylabel('값') plt.tight_layout() plt.show() print(f"Q1 = {q1:.1f}, 중위수 = {median:.1f}, Q3 = {q3:.1f}") print(f"IQR = {iqr:.1f}") print(f"하한 울타리 = {lower_fence:.1f}, 상한 울타리 = {upper_fence:.1f}") print(f"이상치 값 = {outlier_value} → {'이상치임' if outlier_value > upper_fence or outlier_value < lower_fence else '이상치 아님'}") print(f"\n평균 = {np.mean(data):.1f}, 중위수 = {median:.1f} (차이: {np.mean(data)-median:.1f})")
💡 해석 가이드
이상치가 커질수록 평균은 크게 증가하지만, 중위수는 거의 변하지 않는다. 이것이 바로 중위수가 로버스트 통계량이라 불리는 이유이다. IQR 규칙에 의한 상한 울타리(약 Q3 + 1.5×IQR)를 넘는 값은 상자그림에서 별도의 점으로 표시된다.
2.1.6 로버스트 통계량(Robust Statistics)

중위수와 IQR은 극단적인 관측치가 그 값에 거의 영향을 미치지 않기 때문에 로버스트 통계량(robust statistics)이라 불린다. 반면, 평균과 표준편차는 극단적인 관측치의 변화에 더 많이 영향을 받는다.

연습문제 2.6 극단값이 평균과 중위수에 미치는 영향
[미션] 아래 코드에서 max_replacement 값을 100, 500, 1000, 10000 등으로 변경하여 평균과 중위수의 변화를 비교하라.
import numpy as np np.random.seed(42) base_data = np.sort(np.random.normal(50, 10, 100)) # ★ 최댓값을 변경해 보라 ★ max_replacement = 500 # 값: 80, 100, 500, 1000, 10000 scenarios = { '원본': base_data.copy(), f'최댓값→{max_replacement}': np.append(base_data[:-1], max_replacement) } print(f"{'시나리오':<20} {'평균':>10} {'중위수':>10} {'표준편차':>10} {'IQR':>10}") print("-" * 65) for name, d in scenarios.items(): q1, q3 = np.percentile(d, 25), np.percentile(d, 75) print(f"{name:<20} {np.mean(d):10.1f} {np.median(d):10.1f} {np.std(d,ddof=1):10.1f} {q3-q1:10.1f}") print(f"\n관찰: 최댓값을 {max_replacement}(으)로 변경했을 때,") print(f" 평균 변화: {np.mean(scenarios[f'최댓값→{max_replacement}']) - np.mean(scenarios['원본']):.1f}") print(f" 중위수 변화: {np.median(scenarios[f'최댓값→{max_replacement}']) - np.median(scenarios['원본']):.1f}")
💡 해석 가이드
최댓값을 10,000으로 변경하면 평균은 약 100 이상 증가하지만, 중위수는 거의 변하지 않는다(0~1 정도 차이). 이는 중위수가 관측값의 순위에만 의존하고, 평균은 모든 관측값의 실제 크기에 영향을 받기 때문이다.
2.1.7 데이터 변환(Data Transformation) — 특별 주제

데이터가 매우 치우쳐 있거나 변수 간 관계가 비선형일 때, 변환(transformation)을 적용하면 패턴이 더 명확해질 수 있다. 대표적인 변환으로는 로그 변환(log₁₀), 제곱근 변환(√), 역수 변환(1/x) 등이 있다.

연습문제 2.7 로그 변환의 효과 확인하기
[미션] transform을 'log', 'sqrt', 'reciprocal'로 변경하여 변환 전후의 히스토그램을 비교하라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) # 치우친 데이터 (인구 시뮬레이션) population = np.random.exponential(500000, 1000) population = np.clip(population, 1000, 10000000) # ★ 변환 유형을 변경해 보라 ★ transform = 'log' # 'log', 'sqrt', 'reciprocal' fig, axes = plt.subplots(1, 2, figsize=(12, 5)) # 원본 axes[0].hist(population/1e6, bins=30, edgecolor='black', alpha=0.7, color='steelblue') axes[0].set_xlabel('인구 (백만 명)') axes[0].set_title('원본 데이터 (강한 오른쪽 치우침)') axes[0].set_ylabel('빈도') # 변환된 데이터 if transform == 'log': transformed = np.log10(population) xlabel = 'log₁₀(인구)' elif transform == 'sqrt': transformed = np.sqrt(population) xlabel = '√(인구)' elif transform == 'reciprocal': transformed = 1.0 / population * 1e6 xlabel = '1/인구 (×10⁶)' axes[1].hist(transformed, bins=30, edgecolor='black', alpha=0.7, color='coral') axes[1].set_xlabel(xlabel) axes[1].set_title(f'{transform} 변환 후') axes[1].set_ylabel('빈도') plt.tight_layout() plt.show() from scipy.stats import skew print(f"원본 치우침: {skew(population):.2f}") print(f"변환 후 치우침: {skew(transformed):.2f}") print(f"\n변환이 치우침을 {'감소' if abs(skew(transformed)) < abs(skew(population)) else '증가'}시켰다.")
2.1.8 데이터 매핑(Intensity Map) — 특별 주제

지리적 데이터를 만날 때는 색상을 사용하여 변수의 높은 값과 낮은 값을 보여주는 강도 지도(intensity map)를 만들어야 한다. 강도 지도는 특정 위치의 정확한 값을 얻는 데는 유용하지 않지만, 지리적 추세를 파악하는 데는 매우 유용하다.

새로운 시각: 토블러의 지리학 제1법칙

"모든 것은 다른 모든 것과 관련되어 있지만, 가까운 것들이 먼 것들보다 더 관련되어 있다." — 월도 토블러 (Waldo Tobler, 1970)

이 원리는 공간적 자기상관(spatial autocorrelation)이라 불리며, 왜 강도 지도가 유용한지 설명한다. 인접 지역들이 유사한 색상을 가지면 지역적 패턴이 존재하고, 특정 지역이 뚜렷하게 다르면 이상치 또는 특수 요인의 존재를 시사한다.

연습문제 2.8 강도 지도 시뮬레이션
[미션] correlation_strength를 0.0, 0.3, 0.7, 1.0으로 변경하여 두 변수의 공간 상관이 지도에서 어떻게 나타나는지 관찰하라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) rows, cols = 20, 30 # ★ 상관 강도를 변경해 보라 ★ correlation_strength = 0.7 # 0.0 ~ 1.0 # 빈곤율 시뮬레이션 poverty = np.random.uniform(5, 25, (rows, cols)) poverty[15:, :10] += 10 # 남서부 클러스터 # 실업률 시뮬레이션 (빈곤율과 상관) unemployment = correlation_strength * poverty * 0.4 + \ (1 - correlation_strength) * np.random.uniform(2, 10, (rows, cols)) + \ np.random.normal(0, 1, (rows, cols)) fig, axes = plt.subplots(1, 2, figsize=(14, 5)) im1 = axes[0].imshow(poverty, cmap='YlOrRd', aspect='auto') axes[0].set_title('빈곤율 (%)') plt.colorbar(im1, ax=axes[0]) im2 = axes[1].imshow(unemployment, cmap='YlOrRd', aspect='auto') axes[1].set_title('실업률 (%)') plt.colorbar(im2, ax=axes[1]) plt.suptitle(f'강도 지도 시뮬레이션 (상관 강도 = {correlation_strength})') plt.tight_layout() plt.show() corr = np.corrcoef(poverty.flatten(), unemployment.flatten())[0, 1] print(f"빈곤율-실업률 상관계수: {corr:.3f}")

import numpy as np import pandas as pd import matplotlib.pyplot as plt # ★ 아래 분할표 값을 변경해 보라 ★ individual = [3496, 3839, 1170] # 임대, 주택담보, 자가 joint = [362, 950, 183] # 임대, 주택담보, 자가 categories = ['임대(rent)', '주택담보(mortgage)', '자가(own)'] # 분할표 출력 df = pd.DataFrame({'individual': individual, 'joint': joint}, index=categories) df['합계'] = df.sum(axis=1) df.loc['합계'] = df.sum() print("=== 분할표 ===") print(df) # 막대그래프 fig, axes = plt.subplots(1, 2, figsize=(12, 5)) totals = [i + j for i, j in zip(individual, joint)] # 빈도 axes[0].bar(categories, totals, color='steelblue', edgecolor='black') axes[0].set_ylabel('빈도') axes[0].set_title('빈도 막대그래프') # 비율 grand_total = sum(totals) props = [t/grand_total for t in totals] axes[1].bar(categories, props, color='steelblue', edgecolor='black') axes[1].set_ylabel('비율') axes[1].set_title('비율 막대그래프') for i, v in enumerate(props): axes[1].text(i, v + 0.01, f'{v:.3f}', ha='center') plt.tight_layout() plt.show()
2.2.2 행 비율과 열 비율(Row & Column Proportions)

분할표에서 한 변수의 분포를 다른 변수의 수준별로 비교하기 위해, 행 비율 또는 열 비율을 계산할 수 있다. 열 비율이 행에 따라 다르면(또는 그 반대), 두 변수는 연관(associated)되어 있다는 증거이다.

새로운 시각: 설명변수 vs 반응변수

두 변수의 관계를 탐구할 때, 설명변수(explanatory variable)는 결과에 영향을 미칠 것으로 생각되는 변수이고, 반응변수(response variable)는 측정하거나 예측하려는 결과이다.

설명변수에 조건화하여 반응변수의 분포를 확인하는 것이 분석의 기본 원칙이다.

연습문제 2.10 행 비율과 열 비율 계산하기
[미션] 아래 분할표의 값을 변경하고 행 비율과 열 비율을 비교하라. 두 변수 간 연관성이 있는 경우와 없는 경우를 만들어 보라.
import numpy as np import pandas as pd # ★ 아래 분할표 값을 변경해 보라 ★ # 연관 있는 경우 data = np.array([ [209, 158], # 스팸: [텍스트, HTML] [986, 2568] # 비스팸: [텍스트, HTML] ]) # 연관 없는 경우를 시도하려면: # data = np.array([[300, 600], [900, 1800]]) df = pd.DataFrame(data, columns=['텍스트', 'HTML'], index=['스팸', '비스팸']) print("=== 원본 분할표 ===") print(df) # 행 비율 row_props = df.div(df.sum(axis=1), axis=0) print("\n=== 행 비율 ===") print(row_props.round(3)) # 열 비율 col_props = df.div(df.sum(axis=0), axis=1) print("\n=== 열 비율 ===") print(col_props.round(3)) # 연관성 판단 diff = abs(col_props.loc['스팸', '텍스트'] - col_props.loc['스팸', 'HTML']) print(f"\n텍스트 스팸률 vs HTML 스팸률 차이: {diff:.3f}") if diff > 0.05: print("→ 두 변수는 연관되어 있다고 판단할 수 있다.") else: print("→ 두 변수는 독립적(연관 없음)일 수 있다.")

2.3 사례 연구: 말라리아 백신

PfSPZ라는 새로운 말라리아 백신에 대한 연구를 고려하자. 14명의 환자는 백신을, 6명은 위약을 받았다. 19주 후 모두 말라리아 기생충에 노출되었다. 결과는 다음과 같다.

그림 2.29: 말라리아 백신 실험 결과 분할표
감염(infection)무감염(no infection)합계
백신(vaccine)5914
위약(placebo)606
합계11920

백신 그룹의 감염률은 5/14 = 35.7%이고, 위약 그룹의 감염률은 6/6 = 100%이다. 차이는 64.3%포인트이다. 그러나 표본이 작으므로, 이 차이가 백신의 효능 때문인지 우연 때문인지 판단해야 한다.

경쟁하는 두 가설

H₀ (독립 모형): 백신과 감염은 독립적이다. 관찰된 차이 64.3%는 우연에 의한 것이다.

H_A (대립 모형): 백신은 감염률에 영향을 미쳤다. 관찰된 차이는 백신의 효과 때문이다.

2.3.2 연구 시뮬레이션

H₀가 참이라고 가정하고 시뮬레이션을 수행한다. 20명의 환자를 무작위로 14명(백신)과 6명(위약)으로 나누되, 감염 결과(11명 감염, 9명 무감염)는 고정한다. 이 시뮬레이션을 반복하여 우연으로 인한 차이의 분포를 관찰한다.

그림 2.31: 100번 시뮬레이션에서 관찰된 감염률 차이 분포
예제 2.35 시뮬레이션 결과 해석
[문제] 시뮬레이션에서 64.3% 이상의 차이를 얼마나 자주 관찰하는가?
💡 정답 확인
약 2%의 시간에만 발생하는 것으로 나타난다. 이는 드문 사건이므로, H₀를 기각하고 백신이 말라리아에 대해 어느 정도 보호를 제공한다고 결론 내릴 수 있다.
연습문제 2.14 말라리아 백신 시뮬레이션 직접 수행하기
[미션] n_simulations를 100, 1000, 10000으로 변경하여 p-값 추정치의 정밀도가 어떻게 달라지는지 확인하라. 또한 n_vaccine과 n_placebo의 비율을 변경해 보라.
import numpy as np import matplotlib.pyplot as plt np.random.seed(42) # ★ 아래 값을 변경해 보라 ★ n_simulations = 1000 # 시뮬레이션 횟수: 100, 1000, 10000 n_vaccine = 14 # 백신 그룹 크기 n_placebo = 6 # 위약 그룹 크기 n_infected = 11 # 전체 감염자 수 n_not_infected = 9 # 전체 무감염자 수 # 실제 관찰된 차이 observed_diff = n_placebo / n_placebo - 5 / n_vaccine # 1.0 - 0.357 = 0.643 patients = ['infected'] * n_infected + ['not_infected'] * n_not_infected differences = [] for _ in range(n_simulations): np.random.shuffle(patients) vac = patients[:n_vaccine] plac = patients[n_vaccine:] vac_rate = vac.count('infected') / n_vaccine plac_rate = plac.count('infected') / n_placebo differences.append(plac_rate - vac_rate) differences = np.array(differences) plt.figure(figsize=(10, 5)) plt.hist(differences, bins=20, edgecolor='black', alpha=0.7, color='steelblue') plt.axvline(observed_diff, color='red', linewidth=2, linestyle='--', label=f'관찰된 차이 ({observed_diff:.3f})') plt.axvline(0, color='green', linewidth=1, label='기대 차이 (H₀ 하에서 = 0)') plt.xlabel('감염률 차이 (위약 − 백신)') plt.ylabel('빈도') plt.title(f'시뮬레이션 {n_simulations}회 결과') plt.legend() plt.show() p_value = np.mean(differences >= observed_diff) print(f"=== 결과 ===") print(f"관찰된 차이: {observed_diff:.3f} ({observed_diff*100:.1f}%)") print(f"추정 p-값: {p_value:.4f} ({p_value*100:.2f}%)") if p_value < 0.05: print("\np-값 < 0.05 이므로 H₀를 기각한다.") print("→ 백신이 말라리아 감염률을 줄이는 데 효과적이라는 증거가 있다.") else: print("\np-값 ≥ 0.05 이므로 H₀를 기각할 수 없다.")
새로운 시각: 귀무가설 기각의 논리

이 사례 연구는 통계적 추론(statistical inference)의 핵심 아이디어를 보여준다. 이 접근법은 귀류법(proof by contradiction)과 비교할 수 있다:

1. 가정: H₀가 참이라고 가정한다 (백신이 효과 없음).

2. 시뮬레이션: 이 가정 하에서 데이터가 어떻게 보일지 시뮬레이션한다.

3. 비교: 실제 관찰된 결과와 시뮬레이션 결과를 비교한다.

4. 결론: 관찰된 결과가 시뮬레이션에서 드물면 → H₀를 기각한다.

p-값의 직관적 이해: p-값은 "H₀가 참일 때 관찰된 결과(또는 더 극단적인 결과)를 얻을 확률"이다. p-값이 0.05 미만이면 "통계적으로 유의"하다고 간주한다.
